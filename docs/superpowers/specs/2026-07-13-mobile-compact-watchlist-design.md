# 手机端紧凑自选基金设计

## 目标

将基金看板改为以手机为主的纯本地自选列表。首次进入不显示任何默认基金，用户可一次输入多个基金代码，所有基金都能删除。名称与代码合并显示，手机端不再出现横向滚动条。

## 数据模型

- `src/config.ts` 保留 `FUND_LIST` 导出，但数组默认为空，避免出现系统预置基金。
- 用户自选基金存入浏览器 `localStorage`，沿用 `fund-dashboard:custom-funds:v1`。
- 每项保存基金代码、备用名称和分类；估值数据由 Worker 的 `/api/custom-funds` 获取。
- 最多保存 30 只基金，代码必须是六位数字；重复代码不重复添加。
- 本地数据只属于当前浏览器和设备，不跨设备同步。

## 添加与删除

- 页面顶部提供多代码输入框。
- 支持逗号、中文逗号、空格、换行分隔，例如 `012929, 024600`。
- “添加基金”一次提交所有合法且未重复的代码。
- 不提供图片识别。
- 提供“刷新数据”和“一键清空”。
- 每只基金展开后均提供删除按钮，不再区分配置基金和本地基金。
- 一键清空需要二次确认，确认后清除当前浏览器的全部自选基金。

## 紧凑展示

- 桌面与手机共用一套紧凑表格逻辑。
- 基金名称与代码合并在第一列：第一行显示名称，第二行显示“代码 · 分类”。
- 手机端默认只显示三列：基金名称/代码、估算涨幅、估算净值。
- 表格宽度跟随视口，不设置导致横向滚动的最小宽度。
- 点击基金行展开详情，包含昨日净值、更新时间、状态和删除按钮。
- 默认按估算涨幅从高到低排序，涨为红色、跌为绿色。
- 分类按钮、搜索、60 秒刷新和过期提示继续保留。

## 空状态与错误处理

- 首次进入或一键清空后显示“暂无基金，请输入基金代码开始监控”。
- 单个代码查询失败不影响其他基金，失败项仍保留并显示 `--`。
- 网络失败时保留当前页面已展示的数据。
- 无效代码、重复代码和超过数量限制时在输入区域显示明确提示。

## 验证

- TypeScript 类型检查通过。
- 前端 JavaScript 语法检查通过。
- 验证首次打开为空列表。
- 验证逗号、空格和换行批量添加。
- 验证单只删除和一键清空。
- 在约 390px 宽度下验证没有横向滚动条，名称、代码、涨幅和估算净值均可见。
