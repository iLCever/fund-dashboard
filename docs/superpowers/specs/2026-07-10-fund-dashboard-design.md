# 基金盘中估值看板设计

## 目标与范围

创建一个部署到 Cloudflare Workers 免费版的个人基金盘中估值网站。第一阶段使用 TypeScript Worker、Workers Static Assets 和原生 HTML/CSS/JavaScript，提供完整可运行的模拟数据版本。

本阶段不接入真实基金接口、KV、D1、Cron、React、Vue 或传统服务器。

## 产品体验

页面采用手机优先的柔和深蓝风格。手机端基金以卡片显示，基金名称和代码位于首行，盘中估算涨幅使用低饱和色块突出；桌面端使用完整数据表格，窄屏表格区域仍支持横向滚动。

页面顶部包含标题、更新时间、基金搜索框、“添加基金”按钮和自动生成的分类按钮。分类按钮来自配置中的分类值。

“添加基金”在手机端打开底部选择面板，在桌面端打开居中对话框。面板列出 `src/config.ts` 中尚未加入个人列表的候选基金，并支持按名称或代码搜索和添加。已添加基金可从个人列表中移除。

## 基金配置与分类

`src/config.ts` 是候选基金目录的唯一代码配置来源。每只基金包含：

- `code`：基金代码。
- `name`：基金名称。
- `category`：用于顶部筛选的个人分类，例如指数型、混合型、债券型、QDII、黄金商品。
- `fundType`：更准确的产品类型，例如“股票型指数基金 · ETF 联接（QDII）”或“基金中基金 · QDII-LOF-FOF”。
- 模拟估值所需的基础参数。

首批示例基金覆盖五类：

- 015299 华夏纳斯达克100ETF发起式联接（QDII）A：QDII；股票型指数基金 / ETF 联接（QDII）。
- 161116 易方达黄金主题（QDII-LOF-FOF）A：黄金商品；基金中基金 / QDII-LOF-FOF。
- 001180 广发医药卫生联接A：指数型；股票指数型 / ETF 联接。
- 260108 景顺长城新兴成长混合A：混合型；混合型。
- 006544 平安惠聚纯债：债券型；纯债债券型。

产品类型只用于信息展示，不改变用户自定义分类的筛选逻辑。

## 架构与文件职责

- `src/index.ts`：Worker 路由、统一响应、模拟估值生成和 Static Assets 回退。
- `src/config.ts`：候选基金数组和模拟数据参数。
- `src/types.ts`：基金配置、基金估值和 API 响应类型。
- `public/index.html`：语义化页面骨架、添加基金对话框和可访问性标签。
- `public/style.css`：深蓝主题、手机卡片、桌面表格、横向滚动和响应式布局。
- `public/app.js`：请求 API、搜索、分类、排序、渲染、60 秒刷新和本地个人列表管理。
- `wrangler.jsonc`：Worker 入口、兼容日期和 Static Assets 绑定。

## API 与数据流

`GET /api/funds` 返回完整候选基金目录及本次请求生成的模拟估值。每次响应中的基金在各自分类内按估算涨幅从高到低排序。

`GET /api/health` 返回服务状态、当前时间和模拟数据标识。

成功响应统一为：

```json
{
  "success": true,
  "message": "ok",
  "data": {}
}
```

前端启动时请求 `/api/funds`，读取 `localStorage` 中已选择的基金代码，过滤并渲染个人列表。若本地尚无记录，则使用配置中的 `defaultSelected` 字段建立初始个人列表，并保留未选基金供“添加基金”功能使用。用户添加或移除基金后立即更新 `localStorage` 和界面。

页面每 60 秒重新请求一次 API。分类或搜索只在前端处理，不触发额外网络请求。

## 本地存储边界

个人基金代码保存在当前浏览器当前站点的 `localStorage`。刷新、关闭浏览器或设备重启后仍保留，但不会跨浏览器或跨设备同步；无痕模式结束、清除站点数据或更换部署域名后可能丢失。

本地存储只保存基金代码，不保存估值数据。估值始终来自 Worker API。

## 错误与空状态

- API 请求失败时保留上一次成功数据，并显示非阻断错误提示。
- 首次请求失败时显示重试按钮。
- 搜索无结果、当前分类无基金、个人列表为空时分别显示清晰的空状态。
- 不存在的 API 路由返回统一 JSON 错误格式和 404 状态。
- 非 GET API 请求返回 405，并包含 `Allow: GET`。

## 验证

- 运行 `npm install`。
- 运行 `npm run typecheck` 并修复全部 TypeScript 错误。
- 运行 Wrangler dry-run，验证 Worker 与 Static Assets 配置可构建。
- 本地检查两个 API 的状态码、响应结构和排序。
- 检查搜索、分类、添加、移除、刷新保留和 60 秒重载逻辑。
- 检查 320px 手机宽度、桌面宽度和表格横向滚动。

## 完成标准

项目包含用户指定的全部基础文件，能够通过 `npm run dev` 本地启动，通过类型检查和 Worker 构建验证，并且不包含 KV、Cron 或真实基金接口。
